================================================================================
KEY SEPARATIONS:
================================================================================

1. SIMULATION LAYER (abhedya/simulation/)
   - Strictly simulation - no real hardware interfaces
   - Isolated sensor simulation and fusion

2. ANALYSIS LAYER (abhedya/analysis/)
   - Analysis only - no decision execution
   - Threat assessment and classification

3. ADVISORY LAYER (abhedya/advisory/)
   - ADVISORY ONLY - NO EXECUTION AUTHORITY
   - Recommendation generation (non-binding)

4. INTERFACE LAYER (abhedya/interface/)
   - Mandatory human interaction
   - Cannot be bypassed

5. INFRASTRUCTURE LAYER (abhedya/infrastructure/)
   - Cross-cutting concerns
   - Supports all layers with audit, config, logging

6. DOMAIN LAYER (abhedya/domain/)
   - Pure domain logic
   - No infrastructure dependencies

7. CORE ORCHESTRATION (abhedya/core/)
   - System coordination only
   - No execution authority
